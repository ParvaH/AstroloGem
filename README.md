## Architecture Overview

AstroloGem follows a **Strict RAG (Retrieval-Augmented Generation)** design:

- **Rules > References**
- Classical texts are *non-executable* unless explicitly mapped to rules
- If a concept is not operationalized, the system must decline to answer
- Outputs are constrained by interpretation and recommendation guardrails

This repository contains:
- Rule files
- Authority hierarchy
- Interpretation constraints
- Sample outputs
- Test prompts

The AI model itself is **not hosted here**.
